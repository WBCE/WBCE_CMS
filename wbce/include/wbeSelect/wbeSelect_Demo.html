A smarter select
for WBCE CMS

wbeSelect transforms native <select> elements into rich, searchable, icon-aware dropdowns — with tree views, multiselect tags, and a dark/light theme built in. No other dependencies.

Getting Started

Installation

Three includes — stylesheet, jQuery, plugin script. Then call .wbeSelect(), or add class="wbeSelect" for auto-init.

<!-- 1. Stylesheet -->
<link rel="stylesheet" href="wbeSelect.css">

<!-- 2. jQuery (3.x or 4.x) -->
<script src="jquery.min.js"></script>

<!-- 3. Plugin -->
<script src="wbeSelect.jquery.js"></script>
// Auto-init: add class="wbeSelect" to your <select>
<select class="wbeSelect">…</select>

// Manual init with options:
$('#mySelect').wbeSelect({
  customSelector: 'cfg-visibility',
  useSearch:      true,
  treeView:       false
});

Example 03

Tree View with Search

Set treeView: true to wrap monospace prefixes in a styled <span> and show depth-dot notation in the selected state. searchFields: 'text right' lets you search by both page title and page-ID badge. Disabled options act as non-selectable parent headings.

Live Demo — type to search by title or ID
Parent Page
Badge colour key — data-class="type-*"
ID type-public — visible to everyone
ID type-private — specific user only
ID type-registered — logged-in users
ID type-hidden / type-none — not accessible
JS
$('#ap_parent')
  .wbeSelect({
    customSelector: 'flag-right',
    useSearch:      true,
    treeView:       true,
    searchFields:   'text right'
  });
<!-- Single-space prefixes are key!
     HTMLOptionElement.text normalizes
     multi-spaces → single space.
     data-prefix must match exactly. -->

<option
  value="50"
  data-prefix="│ │ ├─ "
  data-level="2"
  data-right="50"
  data-class="type-public"
>│ │ ├─ Tief — Seite X</option>

<!-- Disabled = non-selectable
     root / parent heading -->
<option
  value="6"
  data-level="0"
  data-class="type-public"
  disabled
>Willkommen</option>

Reference

Options

Pass to .wbeSelect({…}) or via data-wbe-select-* attributes on the element.

Option Type Default Description
customSelectorstring''Extra CSS class on the container for scoped styling: 'cfg-visibility', 'flag-right', 'icon_left', 'title_only'.
useSearchbooleantrueShow a search input. Auto-disabled when option count ≤ disable_search_threshold.
disable_search_thresholdnumber7Hide search when total option count is at or below this value. Set to 0 to always show search.
treeViewbooleanfalseEnhances options with data-prefix/data-level into a visual hierarchy.
searchFieldsarray|string['value','text']Which data fields to search. Pass array or space-separated string: 'text right'.
placeholderstring''Empty-state text. Also reads from data-placeholder on the element.
showAllOptionsOnFocusbooleanfalseOpen dropdown on focus before typing (useful in multiple mode).
selectFirstOptionOnSearchbooleantrueAuto-highlight first match when typing in multiple mode.
heightstring'auto'Set to 'element' to lock container to the original element's height.

Option Data Attributes

Enrich individual <option> elements (or the <select> itself) with these attributes:

AttributeEffect
data-left="path.svg"Renders an image on the left of both the option row and the selected tag.
data-right="label"Renders a coloured badge. Position depends on customSelector — right side when using flag-right.
data-subtitle="…"Secondary text line shown in the dropdown below the title. Hidden in the selected state.
data-class="type-public mid-1"Extra classes copied to both the dropdown <li> and selected tag — used for badge colours and left-border accents.
data-prefix="│ ├─ "Tree prefix string. When treeView: true, wrapped in a monospace <span class="tree-prefix">.
data-level="1"Numeric depth. Selected tag shows dot-notation (●●) for level > 0.
data-search-label="…"On the <select>: custom placeholder text inside the search input.
data-placeholder="…"On the <select>: empty-state placeholder text.

Public API

Access the plugin instance via $el.data('wbeSelect') and call these methods:

.refresh()
Re-renders selected item tags after programmatic changes to <option selected> attributes without a full re-init.
.hideDropdown()
Closes the dropdown programmatically — useful on form submit or when navigating away.
.removeSelection( $tagEl? )
Deselects a specific tag (pass the .ws-tag jQuery element), or removes the first selected item when called with no argument.
.destroy()
Removes the widget and restores the original hidden <select> to its visible state.
// Programmatic value change + refresh
var $sel = $('#ap_visibility');
$sel.val('private');
$sel.data('wbeSelect').refresh();

// Destroy and re-init with new options
$sel.data('wbeSelect').destroy();
$sel.wbeSelect({
  customSelector: 'cfg-visibility'
});

CSS Classes & Custom Selectors

Built-in customSelector values that ship with wbeSelect.css:

SelectorEffect
cfg-visibilitySizes left icons to 28px with adjusted top margin — designed for page-visibility icon sets.
flag-rightFloats the right badge to the actual right side of each row — designed for page-tree page-ID numerics.
icon_leftCompact icon layout: 22px left image with adjusted title margin.
title_onlyRemoves left-icon margin — for text-only dropdowns with no images.
pageTreeComboboxFull-width tree combobox with a static inline search input (no absolute positioning).
